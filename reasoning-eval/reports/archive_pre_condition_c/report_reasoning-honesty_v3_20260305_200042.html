# Classifiers in Machine Learning ## Core Concept A classifier is a model that learns to map input features to discrete categories. In the binary case, it learns patterns from training data, then predicts whether new examples belong to class A or class B. **How it works:** 1. During training, the model sees examples with known labels 2. It identifies patterns in features that correlate with each class 3. For new data, it applies these learned patterns to make predictions Think of it like learning to identify spam emails: the model learns which word combinations, sender patterns, and formatting cues predict "spam" vs. "legitimate." ## The Error Tradeoff Here's where it gets interesting. Every classifier has a **decision threshold** — a confidence level above which it predicts "positive." Moving this threshold creates a fundamental tradeoff: | Threshold | Effect | |-----------|--------| | **Lower** | More positive predictions → more false positives (incorrectly flagging negatives) | | **Higher** | Fewer positive predictions → more false negatives (missing actual positives) | **Concrete example:** A cancer screening classifier - **Lower threshold**: Catches more cancers but sends many healthy people for unnecessary biopsies - **Higher threshold**: Fewer false alarms but risks missing actual cancers ## Which Error Matters More? This depends entirely on context: - **False positives are costly**: Spam filters (annoying but recoverable), loan approvals (financial risk) - **False negatives are costly**: Disease detection (health risk), fraud detection (financial loss), security systems (safety risk) **Metrics that capture this:** - **Precision**: Of predicted positives, how many were correct? (minimizes false positives) - **Recall**: Of actual positives, how many did we catch? (minimizes false negatives) - **F1-score**: Harmonic mean balancing both ## Practical Takeaway There's no universally "best" classifier setting. You choose your threshold based on the real-world cost of each error type. A good data scientist explicitly considers: *What does it cost if we're wrong?*

# Angular Resolution and Object Detection ## The Core Concept Angular resolution describes the smallest angular separation between two objects that an optical system can distinguish. It's fundamentally about how finely you can "see" details. ### The Rayleigh Criterion The standard measure comes from the **Rayleigh criterion**, which states two point sources are just resolvable when the central maximum of one diffraction pattern falls on the first minimum of the other. For a circular aperture: $$\theta_{min} = 1.22 \frac{\lambda}{D}$$ Where: - **θ_min** = minimum resolvable angle (in radians) - **λ** = wavelength of light - **D** = aperture diameter ## Connecting Angle to Physical Size Once you know the angular resolution, finding the minimum resolvable object size requires geometry: $$s_{min} = \theta_{min} \times d$$ Where: - **s_min** = minimum object size - **d** = distance to the object ### Practical Example A telescope with 10 cm aperture observing visible light (λ ≈ 550 nm) at 1 km distance: $$\theta_{min} = 1.22 \times \frac{550 \times 10^{-9}}{0.1} ≈ 6.7 \times 10^{-6} \text{ radians}$$ $$s_{min} = 6.7 \times 10^{-6} \times 1000 ≈ 6.7 \text{ mm}$$ ## Key Physical Principles **Diffraction is the limiting factor** — even a perfect lens can't focus light to a point smaller than its wavelength. This creates a fundamental limit independent of lens quality. **Larger apertures win** — doubling aperture diameter halves the minimum resolvable angle, giving you 4× better detail resolution. **Wavelength matters** — shorter wavelengths (UV, X-rays) achieve better resolution than longer ones (infrared, radio). ## Practical Implications | System | Angular Resolution | At 1 km, min size | |--------|-------------------|------------------| | Human eye (5mm pupil) | ~20 arcseconds | ~100 m | | Binoculars (50mm) | ~2 arcseconds | ~10 m | | Hubble (2.4m mirror) | ~0.05 arcseconds | ~0.25 m | This is why astronomical telescopes are so large—they're fighting diffraction to see distant, tiny objects.
