# How the x86 Call Stack Works ## Basic Stack Structure The stack grows **downward** (from high to low memory addresses) on x86. The `ESP` (or `RSP` on x64) register points to the top of the stack. ``` High Memory ┌─────────────────┐ │ Local Vars │ │ (Function B) │ ├─────────────────┤ │ Return Address │ ← pushed by CALL ├─────────────────┤ │ Local Vars │ │ (Function A) │ ├─────────────────┤ │ Return Address │ ├─────────────────┤ │ Arguments │ Low Memory └─────────────────┘ ``` ## The CALL Instruction When you execute `CALL function_name`: ```asm CALL 0x401000 ``` The CPU automatically: 1. **Pushes** the return address (next instruction after CALL) onto the stack 2. **Jumps** to the function ```asm ; Before CALL ESP = 0x1000 ; CALL pushes return address PUSH 0x401005 ; (implicit) ESP = 0x0FFC ; ESP decremented by 4 (on 32-bit) ; Then jumps to function JMP 0x401000 ``` ## Function Prologue When a function starts, it typically: ```asm function_start: PUSH EBP ; Save old base pointer MOV EBP, ESP ; Set up new frame pointer SUB ESP, 0x20 ; Allocate 32 bytes for local variables ``` **After prologue:** ``` ESP → ┌──────────────────┐ │ Local Vars │ (32 bytes) ├──────────────────┤ EBP → │ Old EBP │ ← PUSH EBP saved this ├──────────────────┤ │ Return Address │ ← CALL pushed this ├──────────────────┤ │ Arguments │ └──────────────────┘ ``` ## Accessing Stack Data With `EBP` as the frame pointer, you can access everything: ```asm MOV EAX, [EBP + 8] ; First argument (after return addr) MOV EBX, [EBP + 12] ; Second argument MOV ECX, [EBP - 4] ; First local variable MOV EDX, [EBP - 8] ; Second local variable ``` ## Function Epilogue (RET) When returning: ```asm MOV ESP, EBP ; Deallocate local variables POP EBP ; Restore old base pointer RET ; Pop return address and jump to it ``` The `RET` instruction: 1. **Pops** the return address from the stack 2. **Jumps** to that address ```asm POP EIP ; (implicit) JMP EIP ``` ## Complete Example ```c int add(int a, int b) { int result = a + b; return result; } int main() { int x = add(5, 3); return 0; } ``` **Assembly:** ```asm main: PUSH EBP ; Save old frame MOV EBP, ESP SUB ESP, 4 ; Space for local var 'x' PUSH 3 ; Push second argument PUSH 5 ; Push first argument CALL add ; Push return address, jump to add ADD ESP, 8 ; Clean up arguments MOV [EBP - 4], EAX ; Store result in 'x' MOV ESP, EBP POP EBP RET add: PUSH EBP MOV EBP, ESP SUB ESP, 4 ; Space for 'result' MOV EAX, [EBP + 8] ; Load 'a' ADD EAX, [EBP + 12] ; Add 'b' MOV [EBP - 4], EAX ; Store in 'result' MOV EAX, [EBP - 4] ; Return value in EAX MOV ESP, EBP POP EBP RET ``` ## Key Points | Aspect | Details | |--------|---------| | **Growth** | Downward (high → low addresses) | | **CALL** | Pushes return address automatically | | **RET** | Pops return address and jumps | | **EBP** | Frame pointer (base of current frame) | | **ESP** | Stack pointer (top of stack) | | **Prologue** | Save EBP, set up new frame, allocate locals | | **Epilogue** | Restore ESP, restore EBP, return | This mechanism allows nested function calls to maintain separate local variable spaces and return to the correct location.

# Compilers and Linkers: From Code to Executables ## The Compilation Process ### 1. **Preprocessing** ```c #include <stdio.h> #define MAX 100 int main() { int arr[MAX]; // MAX gets replaced with 100 return 0; } ``` - Expands macros (`#define`) - Includes header files (`#include`) - Removes comments - Output: preprocessed source code ### 2. **Lexical Analysis (Tokenization)** ``` int x = 5; ↓ [int] [x] [=] [5] [;] ``` Breaks code into tokens (keywords, identifiers, operators, literals) ### 3. **Syntax Analysis (Parsing)** Creates an Abstract Syntax Tree (AST): ``` Assignment / | \ int x 5 ``` Validates grammar rules ### 4. **Semantic Analysis** - Type checking - Variable scope verification - Function signature validation ### 5. **Intermediate Code Generation** ```c // Original int result = (a + b) * c; Pops the return address from the stack 2.
